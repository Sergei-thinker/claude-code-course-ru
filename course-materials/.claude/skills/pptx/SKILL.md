# Навык создания PowerPoint-презентаций

Вы можете создавать профессиональные PowerPoint-презентации (.pptx файлы) из markdown-документов. Этот навык разработан для помощи продакт-менеджерам в преобразовании их письменной работы в слайд-деки, готовые для руководства.

## Возможности

- Конвертация markdown-документов в PowerPoint-презентации
- Создание подходящих макетов слайдов на основе структуры контента
- Применение профессионального форматирования и дизайна
- Генерация титульных слайдов, слайдов с контентом и итоговых слайдов
- Поддержка стратегических презентаций, PRD и аналитических документов

## Когда использовать этот навык

Используйте этот навык, когда пользователям нужно:
- Создать презентацию для руководства из стратегического документа
- Преобразовать письменный анализ в слайд-дек
- Представить PRD или спецификации фич стейкхолдерам
- Сгенерировать профессиональные слайды для ревью руководства

## Как создавать презентации

При создании презентации следуйте этому рабочему процессу:

### 1. Анализ исходного документа

Прочитайте markdown-документ и определите:
- Основные разделы, которые станут слайдами
- Ключевые тезисы и подробности
- Визуальную иерархию и поток
- Контент для executive summary

### 2. Проектирование структуры слайдов

Спланируйте структуру презентации:
- **Титульный слайд**: Название документа, дата, контекст
- **Executive Summary**: Ключевые выводы (1-2 слайда)
- **Основные слайды**: Один слайд на каждый крупный раздел
- **Детальные слайды**: Вспомогательная информация, разбитая для читаемости
- **Заключительные слайды**: Итоги, следующие шаги или призывы к действию

### 3. Создание презентации

Используйте Python с библиотекой `python-pptx` для генерации .pptx файла:

```python
from pptx import Presentation
from pptx.util import Inches, Pt
from pptx.enum.text import PP_ALIGN

# Создание объекта презентации
prs = Presentation()
prs.slide_width = Inches(10)
prs.slide_height = Inches(7.5)

# Определение переиспользуемых макетов
def add_title_slide(prs, title, subtitle):
    slide = prs.slides.add_slide(prs.slide_layouts[0])  # Титульный макет
    slide.shapes.title.text = title
    slide.placeholders[1].text = subtitle
    return slide

def add_content_slide(prs, title, content_items):
    slide = prs.slides.add_slide(prs.slide_layouts[1])  # Макет с заголовком и контентом
    slide.shapes.title.text = title

    # Добавление контента как буллет-поинтов
    body = slide.placeholders[1].text_frame
    for item in content_items:
        p = body.add_paragraph()
        p.text = item
        p.level = 0
    return slide

def add_section_slide(prs, section_title):
    slide = prs.slides.add_slide(prs.slide_layouts[2])  # Макет заголовка раздела
    slide.shapes.title.text = section_title
    return slide

# Пример: создание слайдов
add_title_slide(prs, "Название стратегии", "Дата презентации")
add_section_slide(prs, "Раздел 1: Диагностика")
add_content_slide(prs, "Ключевые находки", [
    "Находка 1: Описание здесь",
    "Находка 2: Описание здесь",
    "Находка 3: Описание здесь"
])

# Сохранение презентации
prs.save('output.pptx')
```

### 4. Руководство по форматированию

Применяйте профессиональное форматирование:

**Типографика:**
- Шрифты заголовков: 44pt для заголовков слайдов, 54pt для заголовков разделов
- Основной текст: 18-24pt для основного контента, 16pt для деталей
- Используйте шрифты без засечек (Calibri, Arial или Helvetica)

**Макет:**
- Максимум 3-5 буллет-поинтов на слайд
- Используйте 2-колоночные макеты для сравнений
- Разбивайте плотный контент на несколько слайдов
- Оставляйте свободное пространство для читаемости

**Принципы контента:**
- Одна ключевая идея на слайд
- Используйте активный, лаконичный язык
- Буллеты должны быть параллельны по структуре
- Включайте номера слайдов для ссылок

**Специальные типы слайдов:**

*Стратегические слайды:*
- Структура: Диагностика → Руководящая политика → Действия
- Используйте визуальную иерархию для показа связей
- Выделяйте компромиссы и ключевые решения

*Слайды roadmap:*
- Временная шкала по кварталам/месяцам
- Группируйте связанные инициативы
- Отмечайте зависимости

*Слайды метрик:*
- Текущие vs целевые показатели
- Используйте простые таблицы или графики
- Включайте критерии успеха

## Вывод

После создания презентации:
1. Сохраните .pptx файл с описательным именем
2. Подтвердите расположение файла пользователю
3. Резюмируйте структуру слайдов (количество слайдов и ключевые разделы)
4. Отметьте контент, который был упрощён или реструктурирован

## Лучшие практики

- **Для руководства в первую очередь**: Начинайте с итоговых слайдов, которые руководители могут прочитать за 2 минуты
- **Сканируемость**: Каждый слайд должен быть понятен за 10 секунд
- **Действенность**: Завершайте чёткими следующими шагами или необходимыми решениями
- **Профессионализм**: Используйте единообразное форматирование
- **Учёт контекста**: Адаптируйте формальность и детализацию под аудиторию
