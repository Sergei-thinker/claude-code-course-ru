# Обзор конкурента: Linear

**Последнее обновление:** Октябрь 2024
**Аналитик:** Продуктовая команда TaskFlow

---

## Обзор

Linear — любимый инструмент управления проектами для инженерных команд, основанный в 2019 году бывшими инженерами Airbnb. Они привлекли $50M+ и быстро растут, фокусируясь на скорости, дизайне и опыте разработчиков. Linear представляет "новое поколение" инструментов управления проектами — быстрые, красивые, ориентированные на клавиатуру.

Их основная философия: инструменты управления проектами должны быть такими же быстрыми и отполированными, как продукты, которые создают команды. Это глубоко резонирует с инженерными командами, уставшими от медленных, раздутых инструментов. Исполнение Linear исключительно — они установили новый стандарт производительности и качества дизайна.

Однако их фокус на инженерах создаёт ограничения. Нетехнические члены команды (PM, дизайнеры, маркетологи) находят Linear менее интуитивным. Это наша возможность.

---

## Ключевые функции

**Основные возможности:**
- Отслеживание задач с молниеносной производительностью
- Горячие клавиши для всего (навигация Cmd+K)
- Циклы (2-недельные спринты) и roadmap
- Множественные виды (Список, Доска, Roadmap)
- Пользовательские рабочие процессы и состояния задач
- Вехи проектов и зависимости

**Ориентированность на разработчиков:**
- Интеграция GitHub/GitLab (лучшая в классе)
- API-first архитектура
- CLI инструмент для терминальных рабочих процессов
- Markdown везде
- Командная палитра (Cmd+K)
- Поддержка офлайн

**Заметные преимущества:**
- Загрузка страниц менее 100мс (самая быстрая в категории)
- Красивый, минималистичный дизайн
- Отличная навигация с клавиатуры
- Нативные десктопные приложения (Mac, Windows, Linux)
- Любимые разработчиками интеграции

**Отсутствующие/слабые области:**
- Ограниченная кастомизация (опинионизированные рабочие процессы)
- Слабые функции управления ресурсами
- Базовая отчётность (по сравнению с Asana)
- Не отлично для нетехнических пользователей
- Ограниченная мобильная функциональность

## Целевой рынок

**Основной:** Инженерные команды в технологических компаниях
- Стартапы (Series A-C)
- Команды разработки продуктов
- Технические основатели
- 10-200 инженеров

**Вторичный:** Кросс-функциональные продуктовые команды
- Дизайн + Инженерия + Продукт
- Современные технологичные компании
- Команды, ценящие скорость и дизайн

**Не фокусируется на:** Нетехнических командах, корпоративном сегменте (пока), агентствах, креативных командах

**Примечание:** Linear расширяется за пределы чистых инженерных команд в более широкие продуктовые сценарии использования. Это ставит их в прямую конкуренцию с нами.

## Сильные стороны

1. **Производительность:** Самый быстрый инструмент в категории — загрузка менее 100мс, нулевая задержка
2. **Дизайн:** Красивый, минималистичный, внимание к деталям соперничает с Apple
3. **Опыт разработчика:** Горячие клавиши, CLI, API, интеграция GitHub
4. **Любовь к бренду:** Культовые последователи среди инженеров, сильное сарафанное радио
5. **Фокус:** Опинионизированный продукт с чёткими рабочими процессами (уменьшение паралича решений)
6. **Нативные приложения:** Десктопные приложения Mac, Windows, Linux (не Electron)

---

## Слабые стороны

1. **Ориентированность на инженерию:** Не интуитивен для PM, дизайнеров, нетехнических пользователей
2. **Ограниченная кастомизация:** Опинионизированность = менее гибкий, чем конкуренты
3. **Отчётность:** Слабая аналитика и отчётность по сравнению с Asana, Jira
4. **Мобильные устройства:** Мобильное приложение функционально, но ограничено (не их фокус)
5. **Управление ресурсами:** Нет планирования мощностей или балансировки нагрузки
6. **Кросс-функциональность:** Борется, когда присоединяются нетехнические специалисты (разные ментальные модели)

## Сравнение с TaskFlow

**Где выигрывает Linear:**
- **Скорость:** Их производительность непревзойдённая (менее 100мс против нашего менее 1с)
- **Любовь разработчиков:** Инженеры боготворят Linear
- **Навигация с клавиатуры:** Более комплексные горячие клавиши
- **Нативные приложения:** Десктопные приложения против нашего веб-приложения (пока)
- **Исполнение дизайна:** Немного более отполированный (по общему признанию)

**Где выигрывает TaskFlow:**
- **Кросс-функциональность:** Лучше для PM, дизайнеров, маркетинга, операций
- **Async-First:** Создан для удалённой работы; Linear создан для быстрого взаимодействия
- **Богатство контекста:** Наш акцент на контексте > минималистичный подход Linear
- **Гибкость:** Более настраиваемые рабочие процессы
- **Адаптация:** Проще для нетехнических пользователей
- **Управление ресурсами:** У нас есть виды нагрузки, у них нет

**Стратегическое позиционирование против Linear:**
"Создан для всей продуктовой команды, а не только инженеров. Скорость и красота Linear, с контекстом и коллаборацией для всех."

**Ключевой инсайт:** Linear расширяется на нашу территорию (кросс-функциональные команды). Нам нужно защищаться, будучи ЛУЧШЕ для PM и кросс-функциональной работы, сохраняя их планку качества.

**Сценарии победы:**
- Продуктовые команды с нетехническими членами (PM, дизайнеры, маркетологи)
- Команды с удалённой работой, нуждающиеся в асинхронных рабочих процессах и богатом контексте
- Команды, желающие качество дизайна Linear, но более дружественные к PM функции
- Компании, где инженерия — одна часть более широкой продуктовой организации

**Сценарии проигрыша:**
- Чистые инженерные команды с техническими PM
- Команды, приоритизирующие чистую скорость превыше всего
- Организации, уже глубоко внедрённые в Linear
- Компании инструментов разработчиков (домашняя территория Linear)

## Последние обновления и тенденции

**Обновления Q3 2024:**
- Запущена функция Projects (группировка задач между командами)
- Улучшена визуализация roadmap
- Расширенная интеграция со Slack
- Вид Triage для управления входящими задачами
- Интеграция с Notion (связь базы знаний)

**Траектория роста:**
Linear быстро расширяется. Они движутся от "инструмента для инженеров" к "платформе разработки продуктов." Это напрямую угрожает нашему позиционированию.

**Импульс рынка:**
- Привлечено $50M Series B (лидировал Sequoia)
- 5000+ компаний используют Linear
- Сильные сетевые эффекты (команды массово переключаются с Jira)
- Отличное сарафанное радио в технологическом сообществе

**Уровень стратегической угрозы:** ОЧЕНЬ ВЫСОКИЙ

Linear — наш самый опасный конкурент. У них есть:
- Сильное исполнение продукта (такое же хорошее, как у нас)
- Любовь инженеров (трудно вытеснить)
- Импульс и финансирование
- Расширение на нашу территорию

**Наша защита:**
1. Быть лучше для PM и кросс-функциональных команд (они не могут соответствовать нам здесь без отчуждения разработчиков)
2. Подчёркивать async и контекст (их фокус на скорости означает меньшую глубину контекста)
3. Целиться в команды шире, чем чистая инженерия
4. Соответствовать их планке качества везде
5. Выиграть на мобильных устройствах (они слабы здесь)

**Действия для TaskFlow:**
- Изучить их дизайн-паттерны и соответствовать качеству
- Убедиться, что наша производительность остаётся конкурентной (менее 1с — минимум)
- Подчёркивать сильные стороны PM и кросс-функциональности в позиционировании
- Строить функции, которые они никогда не будут приоритизировать (управление ресурсами, богатый контекст)
- Выиграть мобильные устройства до них
