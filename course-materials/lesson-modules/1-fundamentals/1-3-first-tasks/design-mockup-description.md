ОБЗОР:

Елена представила финальные макеты темной темы для TaskFlow. Это версия 3 после инкорпорирования фидбэка от команд разработки и продукта. Дизайн покрывает основные экраны приложения: проектную доску, детальный вид таска, навигацию и настройки.

ЦВЕТОВАЯ ПАЛИТРА:

Цвета фона:
- Основной фон: #1a1a1a (очень темный серый, не чистый черный)
- Вторичный фон: #242424 (чуть светлее для карточек/панелей)
- Третичный фон: #2e2e2e (для hover состояний)

Цвета текста:
- Основной текст: #f0f0f0 (off-white, не чистый белый - снижает напряжение глаз)
- Вторичный текст: #b0b0b0 (серый для менее важного текста)
- Disabled текст: #666666 (более темный серый)

Акцентные цвета:
- Основной бренд (синий): #4d9fff (светлее чем в светлом режиме #2563eb)
- Success (зеленый): #4ade80 (светлее чем в светлом режиме)
- Warning (желтый): #fbbf24 (такой же как в светлом режиме, уже яркий)
- Error (красный): #f87171 (светлее чем в светлом режиме)
- Ссылки: #60a5fa (более светлый синий, отличается от основного бренда)

Границы/Разделители:
- Тонкие границы: #333333 (едва видимые, достаточно для разделения)
- Сильные границы: #444444 (для выделенного разделения)

ПРИМЕЧАТЕЛЬНЫЕ ДИЗАЙН РЕШЕНИЯ:

Коэффициенты контраста:
Елена проверила что весь текст соответствует стандартам WCAG AA (4.5:1 для обычного текста, 3:1 для крупного текста). Основной текст (#f0f0f0 на #1a1a1a) имеет коэффициент 11.5:1. Вторичный текст (#b0b0b0 на #1a1a1a) имеет коэффициент 6.2:1.

Не чистый черный:
"Я использую #1a1a1a вместо #000000 потому что чистый черный на OLED экранах может вызывать напряжение глаз с чистым белым текстом. Чуть более светлый черный легче для глаз."

Подсветка синтаксиса:
Блоки кода в описаниях тасков используют темную тему синтаксиса (похожую на Dark+ тему VS Code). Елена включила примеры с JavaScript, Python и JSON.

Загруженные пользователями изображения:
Изображения во вложениях тасков имеют тонкую границу #333333 для отделения от фона. Елена отметила: "Если пользователи загрузят скриншоты с белыми фонами, они будут выглядеть странно. Граница помогает."

Иконки:
Все иконки #f0f0f0 или #b0b0b0 в зависимости от важности. Елена скорректировала несколько иконок которые были слишком тонкими (выглядели блекло в темном режиме) - сделала их чуть более жирными.

ОПАСЕНИЯ / ВОПРОСЫ ОТ РЕВЬЮ:

1. ЗАГРУЖЕННЫЕ ПОЛЬЗОВАТЕЛЯМИ ИЗОБРАЖЕНИЯ:
Проблема: Скриншоты с белыми фонами будут выглядеть резко
Решение Елены: Добавить границу, но может быть недостаточно
Альтернатива: Добавить фоновый паттерн за изображениями (как делает Slack?)
ТРЕБУЕТСЯ РЕШЕНИЕ: Как это обрабатывать? Тестировать с реальным контентом?

2. ЦВЕТА ГРАФИКОВ/ДИАГРАММ:
Проблема: Страница отчетов имеет графики (velocity, burndown). Елена показала примеры но цвета могут требовать корректировки.
Опасение: Библиотеки графиков (мы используем Recharts) имеют цвета по умолчанию которые не будут работать в темном режиме.
ACTION: Разработке нужно протестировать поддержку темного режима библиотеки графиков

3. СТОРОННИЕ ВСТРАИВАНИЯ:
Проблема: Встраивания Figma, встраивания Google Doc - мы не контролируем их внешний вид
Елена: "Если кто-то встроит Figma макет, он будет в светлом режиме. Мы не можем это изменить."
ТРЕБУЕТСЯ РЕШЕНИЕ: Принять что встроенный контент не будет соответствовать теме? Или скрыть встраивания в темном режиме?

4. МОБИЛЬНОЕ ПРИЛОЖЕНИЕ:
Проблема: Елена проектировала это для web. Mobile требует отдельного рассмотрения.
Вопрос: Запускается ли mobile с темной темой? Или отгрузить mobile без темной темы изначально?
Дмитрий (mobile PM) должен взвесить.

5. МАРКЕТИНГОВЫЙ САЙТ:
Проблема: Должен ли наш маркетинговый сайт (taskflow.com) тоже иметь темную тему?

Елена: "Это дополнительный скоуп. Маркетинговый сайт - отдельная кодовая база."
ТРЕБУЕТСЯ РЕШЕНИЕ: Только приложение или приложение + маркетинговый сайт?

6. РЕЖИМ ПО УМОЛЧАНИЮ:
Проблема: Должна ли темная тема быть opt-in или opt-out?
Опции:
  a) По умолчанию светлый режим (текущее поведение)
  b) По умолчанию системное предпочтение (авто-определение темы ОС)
  c) По умолчанию темный режим (заставить всех на темный)
Елена предлагает опцию B (системное предпочтение)

ЗАМЕТКИ ПО ТЕХНИЧЕСКОЙ РЕАЛИЗАЦИИ:

Елена работала с Jamie (eng lead) над подходом к реализации:

CSS переменные:
Использовать CSS custom properties для цветов:
--background-primary: #1a1a1a;
--text-primary: #f0f0f0;
и т.д.

Переключение между светлой/темной изменением корневых CSS переменных.

Сохранение:
Предпочтение пользователя хранится в базе данных (настройки пользователя)
Также хранится в localStorage для мгновенной загрузки

Анимация:
Плавный переход при переключении (0.2s ease)
Елена: "Никаких резких вспышек между режимами"

Определение системного предпочтения:
Использовать media query prefers-color-scheme
Уважает настройку темного режима на уровне ОС

Переопределения по компонентам:
Некоторые компоненты могут требовать специальной обработки (графики, встраивания)
Разработке определить edge cases во время реализации
