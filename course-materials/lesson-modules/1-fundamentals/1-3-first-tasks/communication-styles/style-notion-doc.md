# Стиль коммуникации: Notion документ

## Цель
Комплексный справочный документ который любой может прочитать асинхронно и получить полный контекст.

## Правила формата
- **Длина:** Столько сколько нужно для основательности (обычно 300-600 слов)
- **Тон:** Профессиональный но доступный, детальный, хорошо организованный
- **Структура:**
  - **Заголовок:** Четкий, описательный headline
  - **Саммари:** 2-3 предложения TL;DR в начале
  - **Секции:** Использовать H2 заголовки для организации (Background, Ключевые находки, Рекомендации, Следующие шаги)
  - **Списки:** Буллет-пойнты для сканируемости
  - **Таблицы:** Опционально для организации данных/находок
  - **Выделения:** Использовать для важных инсайтов или action items
- **Включать:** Полный контекст, поддерживающие детали, прямые цитаты, конкретные примеры
- **Избегать:** Предположение предварительных знаний, пропуск важного контекста
- **Фокус:** Быть комплексным, самостоятельным справочником

## Пример

**Входящие данные:** User research показало что 8/8 пользователей столкнулись с трудностями при онбординге, тратя 45+ минут на завершение первого таска. Основные болевые точки: неясная навигация (6 упоминаний), отсутствие шаблонов (5 упоминаний), подавляющий пустой экран (4 упоминания). Рекомендуется: добавить интерактивный тур и библиотеку шаблонов.

**Выход:**

# Отчет user research: Исследование онбординга TaskFlow

**TL;DR:** Все 8 пользователей значительно столкнулись с трудностями при онбординге (средние 45+ мин до первого таска). Основные проблемы: путаница в навигации, отсутствие шаблонов, подавляющий опыт пустого экрана. Рекомендуем интерактивный тур + библиотеку шаблонов для Q1.

## Background

Мы провели 8 интервью с пользователями (2-6 окт) с новыми пользователями TaskFlow чтобы понять точки трения онбординга. Участники были набраны из недавних trial регистраций и представляли микс ролей (PMs, инженеры, маркетологи) и размеров команд (от solo до 50+ человек).

**Цели исследования:**
- Идентифицировать точки трения в опыте first-time пользователя
- Понять time-to-first-task completion
- Сравнить наш онбординг с опытом конкурентов
- Сгенерировать рекомендации для улучшений активации Q1

## Ключевые находки

### 1. Time-to-Value слишком долгий
- **Среднее время до завершения первого таска: 45 минут**
- Все 8 пользователей (100%) столкнулись с трудностями при первоначальной настройке
- Конкуренты в среднем 15 минут (Linear, Asana)
- Текущий показатель активации: 45% (цель: 60%)

### 2. Навигация запутанная (6/8 пользователей)
Пользователи не могли найти базовые фичи:
- "Я не знал куда кликнуть чтобы создать свой первый проект"
- "Я искал шаблоны но не мог найти их"
- "Сайдбар не был очевиден как навигация"

### 3. Отсутствие шаблонов увеличивает трение (5/8 пользователей)
Пользователи хотят предсобранные стартовые точки:
- "Я уставился на пустой экран не зная что делать"
- "Хотелось бы примеров от которых можно стартовать"
- "Notion имеет шаблоны — почему вы нет?"

### 4. Опыт пустого экрана подавляющий (4/8 пользователей)
Пустое рабочее пространство пугает новых пользователей:
- "Я чувствовал что должен строить всё с нуля"
- "Это слишком много свободы без руководства"

## Рекомендации

### Основная: Интерактивный продуктовый тур
- Провести пользователей через ключевые фичи при первом входе
- Progressive disclosure (не перегружать)
- Можно пропустить но по умолчанию "вкл"
- Оценка разработки: 2 спринта

### Основная: Библиотека шаблонов
- 5-7 предсобранных проектных шаблонов
- Категории: Запуск продукта, Планирование спринта, Отслеживание багов, Контент-календарь
- Показывается сразу после регистрации
- Оценка разработки: 3 спринта (web), 1 спринт (mobile позже)

### Вторичная: Улучшения навигации
- Более выделенные метки сайдбара
- Контекстные подсказки для новых пользователей
- Включено в интерактивный тур
