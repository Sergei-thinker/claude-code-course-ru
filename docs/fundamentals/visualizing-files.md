# 1.2: Визуализация вашего рабочего пространства

- **Время выполнения:** 15-20 минут
- **Требования:** Модуль 1.1 пройден, базовое понимание файловых систем

> **Начните этот модуль в Claude Code:** Запустите `/start-1-2` для начала интерактивного обучения.

## Обзор

Настройте визуальное рабочее пространство для просмотра файлов проекта в реальном времени рядом с Claude Code для бесшовного split-screen workflow.

**Главное:** Визуальное рабочее пространство позволяет наблюдать, как файлы появляются и обновляются, пока Claude работает.

## Почему визуализация важна

Работа с Claude Code в терминале похожа на работу вслепую. Вы просите Claude создать документы, но не видите, что происходит.

**Решение:** Откройте визуальный редактор рядом с терминалом. Теперь вы можете:
- Смотреть, как файлы появляются, пока Claude их создаёт
- Читать документы, пока Claude работает
- Проверять, что всё организовано правильно
- Всегда точно знать, что происходит

Это превращает Claude Code из загадочного чёрного ящика в прозрачный инструмент, который можно наблюдать и проверять в реальном времени.

## Папка .claude/

Папка `.claude/` содержит агентов, команды и настройки. Доступ зависит от вашего инструмента:

| Инструмент | Видна .claude/? | Как получить доступ |
|------|-------------------|---------------|
| **Nimbalyst** | Да | Видна в браузере файлов |
| **Obsidian** | Нет | Используйте Finder/Explorer |
| **VS Code** | Да | Включите скрытые файлы |

**Чтобы просмотреть `.claude/` в Finder/Explorer:**
- **Mac:** Нажмите `Cmd + Shift + .` в Finder, чтобы показать скрытые папки
- **Windows:** Включите "Hidden items" (Скрытые элементы) во вкладке View в File Explorer

**Когда это понадобится:** Модуль 1.5 (Пользовательские Sub-агенты) и позже. Пока сосредоточьтесь на обычных файлах проекта.

**Примечание:** Claude Code всегда может получить доступ к файлам `.claude/` независимо от вашего редактора.

## Демо визуализации в реальном времени

Когда Claude создаёт или редактирует файлы, вы будете наблюдать, как они появляются в вашем редакторе мгновенно.

**Попробуйте:**

1. Откройте редактор с папкой проекта
2. В Claude Code спросите: *Create a file called test-visualization.md with some content*
3. Наблюдайте за редактором - файл появляется немедленно
4. Кликните на него, чтобы прочитать, что написал Claude

Вот и всё! Обновление не требуется. Файлы обновляются в реальном времени, пока Claude работает.

## Советы по редактору

### Советы Nimbalyst
- Следите за выделенными diff, когда Claude редактирует файлы
- Папка `.claude/` видна для создания пользовательских агентов позже

### Советы Obsidian

**Основные горячие клавиши:**
- **`Cmd/Ctrl + O`** - Quick Switcher (переход к любому файлу)
- **`Cmd/Ctrl + Shift + F`** - Поиск по всем файлам
- **`Cmd/Ctrl + E`** - Переключение между режимами Reading/Editing

**Wikilinks:** Соединяйте документы с синтаксисом `[[filename]]` для лёгкой навигации

**Теги:** Организуйте с помощью `#prd #user-research #q1-2025` для фильтрации

**Оставайтесь в режиме Reading** для более чистого форматированного отображения

### Советы VS Code
- **`Cmd/Ctrl + P`** - Быстрое открытие любого файла
- Включите скрытые файлы, чтобы видеть `.claude/`

## Устранение неполадок

### Я не вижу никаких файлов

**Вероятная причина:** Открыта не та папка

**Решение:**
1. Спросите Claude: `What's the path to my course folder?`
2. Откройте именно эту папку в редакторе
3. Ваши файлы должны появиться

**Частая ошибка:** Открытие родительской папки вместо самой папки проекта

**Пример:**
- Неправильно: `/Users/yourname/Documents` (слишком высокий уровень)
- Правильно: `/Users/yourname/Documents/claude-code-pm-course`

### Файлы не обновляются

**Вероятная причина:** Редактору нужно обнаружить изменение

**Решение:**
1. Кликните вне окна редактора и обратно (принудительное обновление)
2. Или: Закройте и откройте конкретный файл заново
3. Или: Перезапустите редактор (редко, но работает)

### Я вижу сырой markdown вместо форматированного текста

**Для Obsidian:** Нажмите `Cmd/Ctrl + E` для переключения в режим Reading, или кликните значок книги

**Для Nimbalyst:** Содержимое должно отображаться форматированным по умолчанию (WYSIWYG)

### Где папка .claude/?

**Nimbalyst:** Должна быть видна в браузере файлов

**Obsidian:** Скрыта по дизайну. Используйте Finder/Explorer:
- **Mac:** Откройте Finder → Перейдите к проекту → Нажмите `Cmd + Shift + .`
- **Windows:** Откройте File Explorer → Перейдите к проекту → Вкладка View → Отметьте "Hidden items"
