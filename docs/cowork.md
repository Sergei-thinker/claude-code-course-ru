## Что такое Cowork?

> **Урок 1:** Скажите Claude *"Read START-HERE.md and start lesson 1"*

**Claude Cowork - это Claude Code для не-разработчиков.** Тот же мощный ИИ агент, обёрнутый в интерфейс, который не требует терминала.

Вы даёте Claude доступ к папке на вашем компьютере. Claude может затем читать, редактировать и создавать файлы в той папке - организовывать загрузки, строить таблицы из чеков, писать отчёты из разрозненных заметок и больше.

**Ключевое отличие от обычного Claude:** Cowork производит настоящие файлы, а не текст, который вы копируете и вставляете. Попросите expense таблицу, получите реальный `.xlsx` с работающими формулами. Попросите презентацию, получите реальный `.pptx` со слайдами.

### Метафора

| Инструмент | Роль | Что делает |
|------|------|--------------|
| **Claude** | Исследователь | Отвечает на вопросы, проводит исследования |
| **Claude Code** | CTO | Пишет и редактирует код |
| **Claude Cowork** | COO | Оперирует вашей рабочей жизнью - файлы, документы, исследования |

Anthropic построил Cowork примерно за 10 дней. Используя Claude Code.

## Фреймворк "Готово"

> **Урок 3:** Скажите Claude *"Read START-HERE.md and start lesson 3"*

Качество вашего вывода напрямую пропорционально качеству вашего ввода.

Перед делегированием чего-либо, ответьте на три вопроса:

### 1. Как выглядит "готово"?

Если вы не можете представить финальное состояние, Claude тоже не может.

| ❌ Расплывчато | ✅ Конкретно |
|----------|-------------|
| "Создай expense отчёт" | "Создай Excel таблицу с колонками для даты, vendor, суммы и категории, отсортированную по дате, с итоговой суммой внизу" |

### 2. Какой контекст нужен Claude?

- Ваши соглашения об именовании?
- Ваши предпочтения структуры папок?
- Разница между рабочими и личными документами?

Если не скажете Claude, она сделает предположения. Иногда правильные. Иногда нет.

### 3. Какие ограничения важны?

- "Не удалять никакие файлы, просто переместить их."
- "Сохранить оригинальные имена файлов, но добавить префикс даты."
- "Обрабатывать только файлы за последние 90 дней."

Claude не может читать ваши мысли о лимитах, которые вы не указали.

**Самый безопасный дефолт:** Всегда говорите "не удалять ничего", если специально не хотите удалений.

```
Прочитай все документы в этой папке. Создай единый отчёт,
который синтезирует ключевые находки. Включи прямые цитаты, где
релевантно, с именем исходного файла. Отметь любые противоречия

```


## Настройка папок

> **Урок 5:** Скажите Claude *"Read START-HERE.md and start lesson 5"*

Не давайте Claude доступ к случайным папкам. Создайте систему.

### Рекомендуемая структура

```
Claude-Work/
├── inbox/          ← Бросайте файлы сюда для обработки Claude
├── processed/      ← Claude перемещает готовую работу сюда
├── outputs/        ← Claude создаёт новые файлы здесь
└── reference/      ← Файлы, которые Claude должен читать, но не модифицировать
```

### Скажите Claude об этом

Включите это в ваш первый промпт:

```
Папка inbox содержит файлы для обработки. Переместить их в
processed, когда готово. Создавать любые новые файлы в outputs.
Папка reference только для чтения - не модифицировать те файлы.
```

Это создаёт повторяемый workflow. Бросить файлы в inbox, сказать Claude обработать, найти результаты в outputs.

## Операции с файлами

> **Уроки 2 и 4:** Скажите Claude *"Read START-HERE.md and start lesson 2"*

Это суперсила Cowork: **content-aware организация файлов**.

### Что делает это другим

Обычная организация файлов смотрит на имена файлов. `IMG_4521.png` идёт в "Images."

Cowork смотрит на **содержимое**. Тот `IMG_4521.png`? Claude может видеть, что это на самом деле чек от coffee поставщика. Идёт в "Receipts/Coffee-Suppliers."

### Масштаб

Люди организовали 300+ файлов за минуты. Папки Downloads с годами хаоса - отсортированы, фактически читая, что каждый файл содержит.

### Паттерн

1. Указать Claude на грязную папку
2. Дать чёткое делегирование с фреймворком Done
3. Запросить change log, чтобы вы могли проверить решения
4. Позволить запуститься

## Исследование и синтез

> **Урок 6:** Скажите Claude *"Read START-HERE.md and start lesson 6"*

Здесь Cowork экономит часы.

### Проблема

У вас 20 документов - интервью клиентов, результаты опросов, тикеты поддержки. Вам нужны паттерны и инсайты.

Вручную: Прочитать каждый документ, делать заметки, перекрёстно ссылаться, писать находки. Часы работы.

### Подход Cowork

Claude читает все документы одновременно и синтезирует через них - не просто суммируя каждый, но соединяя точки между ними.

Клиент A сказал одно. Клиент B сказал что-то связанное. Клиент C противоречил обоим. Claude ловит всё это и цитирует источники.

### Что запрашивать

```
Прочитай все документы в этой папке. Создай synthesis отчёт с:
- Executive summary
- Ключевые темы (с поддерживающими цитатами и цитированием исходных файлов)
- Противоречия между источниками
- Паттерны по [сегменту/категории/типу источника]
- Конкретные [запросы/рекомендации/инсайты] упомянутые
- Вопросы, которые остаются без ответа
```

## Создание документов

> **Урок 8:** Скажите Claude *"Read START-HERE.md and start lesson 8"*

Cowork производит **настоящие файлы**, а не текст, который вы копируете и вставляете.

### Что делает каждый формат

| Формат | Возможности |
|--------|--------------|
| **Excel (.xlsx)** | Работающие формулы (SUM, VLOOKUP, IF), условное форматирование, множественные листы, charts, валидация данных |
| **PowerPoint (.pptx)** | Реальные слайды с layouts, speaker notes, логическим flow - не просто bullet points |
| **Word (.docx)** | Правильные headings, tables of contents, отформатированные таблицы |
| **PDF** | Заполнение форм, сборка документов |

### Пример: Чек → Expense Tracker

```
У меня фото чеков в этой папке. Создай expense-tracker.xlsx с:
- Data sheet: Date, Vendor, Category, Amount, Payment Method
- Формулы для totals по категориям
- Условное форматирование для расходов свыше $100
- Summary sheet с chart
- Flagged items sheet для unclear чеков
```

Вы получаете реальный Excel файл. Открыть, редактировать, делиться им.

## Субагенты

> **Урок 7:** Скажите Claude *"Read START-HERE.md and start lesson 7"*

Когда задача имеет независимые части, Claude запускает множественных workers для их решения одновременно.

### Как это работает

Вместо обработки 10 файлов последовательно (50 минут), Claude обрабатывает все 10 параллельно (5 минут), затем синтезирует результаты.

### Когда явно запрашивать их

| Паттерн | Пример промпта |
|---------|----------------|
| **Анализ vendor** | "Запусти субагентов для исследования каждого из этих четырёх vendors независимо. Затем синтезируй в сравнение." |
| **Мультиперспектива** | "Анализируй это решение, используя трёх субагентов: финансовое влияние, опыт клиента, операционный риск. Синтезируй находки." |
| **Большие наборы документов** | "Используй субагентов для обработки этих 50 транскриптов параллельно. Извлеки темы, затем синтезируй в один отчёт." |

## Автоматизация браузера

> **Урок 9:** Скажите Claude *"Read START-HERE.md and start lesson 9"*

С расширением **Claude in Chrome** Cowork может контролировать ваш браузер.

### Что работает хорошо

| Случай использования | Пример |
|----------|---------|
| **Конкурентное исследование** | Посещать pricing pages, извлекать tiers и функции, создавать comparison таблицу |

Работает отлично иногда. Застревает в другое время. Если Claude застряла, скажите попробовать снова или пропустить ту часть.

## Система Skills

> **Уроки 5 и 11:** Скажите Claude *"Read START-HERE.md and start lesson 5"*

Skills - переиспользуемые наборы инструкций, которым следует Claude.

### Что делают Skills

Вместо переобъяснения ваших предпочтений каждую сессию, создайте skill раз и ссылайтесь навсегда.

### Создание кастомного Skill

Поместите markdown файл в вашу рабочую папку:

```markdown
# Мой гайд стиля отчётов

## Формат
- Всегда использовать bullet points, не параграфы
- Включить executive summary вверху
- Закончить секцией "Next Steps"

## Голос
- Профессионально, но не жёстко
- Без жаргона без объяснения
- Active voice

## Структура для статусных отчётов
1. Summary (макс 3 bullets)
2. Completed This Week
3. In Progress
4. Blockers
5. Next Week
```

Затем скажите Claude: *"Read style-guide.md and follow it for all documents you create."*

## Паттерн ИИ сотрудника

> **Урок 10:** Скажите Claude *"Read START-HERE.md and start lesson 10"*

Это менталитет, который раскрывает полную ценность Cowork.

### Старый способ (мышление chatbot)

```
Дать задачу → Ждать → Получить результат → Дать следующую задачу → Ждать → Получить результат
```

### Способ ИИ сотрудника

```
Выгрузить все задачи → Передать их → Сделать что-то другое → Вернуться к готовой работе
```

### Стратегия пакетирования

Вместо трёх отдельных сессий:

```
"Организуй мои чеки" → ждать → готово
"Создай expense отчёт" → ждать → готово
"Суммируй для моего бухгалтера" → ждать → готово
```

Делай так - одна сессия, один промпт:

```
У меня скриншоты чеков в /receipts. Сначала, организуй их
по месяцам в подпапки. Затем создай expense таблицу
со всеми данными. Наконец, создай одностраничный summary для моего
бухгалтера, показывающий totals по категориям. Поместить все outputs в /outputs.
```

Одна сессия. Три задачи. Меньше использования. Лучшие результаты.

## Cowork vs Claude Code

| Аспект | Claude Code | Cowork |
|--------|-------------|--------|
| **Интерфейс** | Терминал | GUI |
| **Основное использование** | Код/разработка | Всё остальное |
| **Пользователь** | Разработчики | Все |
| **Доступ к файлам** | Рабочая директория | Выбор папки |
| **Кривая обучения** | Выше | Ниже |
| **Кастомизация** | CLAUDE.md, slash команды | Skills |

**Используйте Claude Code, когда:** Пишете код, работаете в кодовой базе, нужен доступ к терминалу.

**Используйте Cowork, когда:** Делаете knowledge работу, организуете файлы, создаёте документы, проводите исследования.

## Чеклист безопасности

### Перед первой сессией

- [ ] Создать выделенную рабочую папку (не Documents или Desktop)
- [ ] Backup любых важных файлов, к которым дадите Claude доступ
- [ ] Решить, что Claude НЕ разрешено удалять

### Каждая сессия

- [ ] Быть явным об ограничениях в вашем промпте
- [ ] Избегать размещения sensitive файлов в рабочей папке (credentials, финансовые документы, личные записи)
- [ ] Если используете browser доступ, придерживаться trusted сайтов

### Золотое правило

**Всегда говорите "не удалять ничего", если специально не хотите удалений.** Claude удаляет только когда проинструктирован, но будьте явными.

## TL;DR

1. **Cowork - worker, не chatbot.** Делегировать результаты, отойти, вернуться к готовым файлам.

2. **Писать лучшие промпты.** Определить, как выглядит "готово", дать контекст, указать ограничения.

3. **Начать с выделенной папки.** Ограничить доступ во время обучения. Держать backups.

4. **Использовать субагентов для сложной работы.** Claude параллелизирует автоматически, когда полезно.

5. **Skills производят реальные файлы.** Excel с формулами, PowerPoint презентации, отформатированные документы.

6. **Ставить в очередь задачи и пакетировать связанную работу** для экономии использования.

7. **Безопасность - ваша ответственность.** Backup файлов, избегать sensitive данных, мониторить, что делает Claude.
