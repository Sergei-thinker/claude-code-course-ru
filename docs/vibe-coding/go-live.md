# 4.5: Go Live

- **Время на выполнение:** 15 минут

> **Интерактивный урок:** Выполните `/start-4-5` в Claude Code

**В этом уроке:** Деплоите ваш проект и получите live URL для sharing.

## Что такое Deployment?

Deployment - это взятие кода с вашего компьютера и размещение в интернете.

Ваш код идёт с вашей машины → серверы, к которым любой может получить доступ → реальный URL.

Разработчики говорят "deploy," "ship," или "go live" постоянно. Это момент, когда ваша вещь становится реальной.

## Что такое Hosting?

Hosting означает серверы, которые запускают ваш код 24/7 и обрабатывают трафик от посетителей.

Вы не управляете серверами самостоятельно - хостинг сервис обрабатывает всё это. Вы просто даёте им ваш код.

## Что такое Vercel?

Vercel - хостинг сервис для веб-сайтов. Это:
- **Бесплатно** для личных проектов (щедрый free tier)
- **Сделано создателями Next.js** - они работают идеально вместе
- **Подключено к GitHub** - auto-deploys, когда вы push изменения

Даёте Vercel ваш GitHub repo → строит ваш код → вы получаете реальный URL вроде `yourproject.vercel.app`.

## Как работает Auto-Deploy

Это магия:

1. Вы push код на GitHub
2. Vercel детектирует изменение
3. Vercel rebuilds и redeploys автоматически
4. Ваш live сайт обновляется за ~1 минуту

Нет ручных шагов после начальной настройки. Изменить код → push → live.

## Что такое Vercel CLI?

Vercel CLI - инструмент командной строки для деплоя напрямую из вашего терминала.

Claude использует его для деплоя без выхода из Claude Code. Аутентифицируется через ваш браузер, похоже на GitHub.

## Устранение неполадок

### "vercel: command not found"
Vercel CLI не установлен. Выполните `npm i -g vercel`

### Аутентификация не удалась
Выполните `vercel login` снова и завершите browser authorization.

### Build не удался
Проверьте Vercel dashboard для error logs. Обычно это проблема кода - скажите Claude, что говорит ошибка, и она может помочь исправить.

### "Project not found"
Убедитесь, что вы в правильной проектной папке, когда выполняете deploy команду.

### Сайт показывает старую версию
Deployment может всё ещё строиться. Проверьте Vercel dashboard для статуса - обычно занимает около минуты.

### Работает локально, но не на Vercel
Различия сред. Проверьте Vercel build logs для подсказок о том, что различно.

## После Deployment

Когда вы live:

1. **Тестировать на телефоне** - тот же URL работает на любом устройстве
2. **Делиться с друзьями** - отправить ссылку, смотреть их реакцию
3. **Будущие обновления:** изменить код → push на GitHub → auto-deploys

Это весь flow. Теперь вы знаете, как строить и ship вещи.

## Полный Loop

Вы узнали:

| Шаг | Что означает |
|------|---------------|
| **План** | Получить ясность о том, что строите |
| **Build** | Позволить ИИ создать это |
| **Итерация** | Уточнять, пока не правильно |
| **Сохранить** | Backup на GitHub |
| **Go Live** | Выложить в интернет |

Этот loop работает для любого проекта. Landing pages, инструменты, приложения, прототипы - тот же процесс.
