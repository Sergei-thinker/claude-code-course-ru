# 4.4: GitHub

- **Время на выполнение:** 15 минут

> **Интерактивный урок:** Выполните `/start-4-4` в Claude Code

**В этом уроке:** Push вашего проекта в private GitHub repository.

## Что такое контроль версий?

Контроль версий - система, которая отслеживает каждое изменение ваших файлов во времени.

Думайте об этом как "отслеживание изменений" в Word, но для целых проектов. Вы можете:
- Видеть историю каждого изменения
- Вернуться к любой предыдущей версии
- Отменить ошибки

Решает проблему "Final_v3_REAL_FINAL.docx" навсегда.

## Git vs GitHub

Это связано, но различно:

**Git** = Система контроля версий, которая работает на вашем компьютере. Отслеживает изменения локально.

**GitHub** = Веб-сайт, который хранит ваши Git repositories в облаке.

Аналогия: Git как Microsoft Word (инструмент). GitHub как Google Drive (где вы храните и делитесь файлами).

## Ключевые термины

| Термин | Что означает |
|------|---------------|
| **Repository (repo)** | Проектная папка, отслеживаемая Git |
| **Commit** | Сохранённый snapshot в момент времени |
| **Push** | Загрузить commits с вашего компьютера → GitHub |
| **Pull** | Скачать изменения с GitHub → ваш компьютер |

## Почему вам это нужно для deployment

Прямо сейчас ваш код существует только на вашем компьютере. Если ваш ноутбук умрёт, он пропал.

Более важно: deployment сервисы нуждаются pull вашего кода откуда-то. GitHub - это то место.

Когда вы деплоите в следующем уроке, Vercel подключится к вашему GitHub repo и pull код автоматически.

## Что такое GitHub CLI?

GitHub CLI (`gh`) - инструмент командной строки для работы с GitHub.

Claude использует его, чтобы вам не нужно было вводить Git команды самостоятельно. Аутентифицируется через ваш браузер, используя device code flow - вы увидите код, введёте его на веб-сайте GitHub, и вы подключены.

## Устранение неполадок

### "gh: command not found"
GitHub CLI не установлен.
- **Mac:** Выполните `brew install gh`
- **Другие:** Загрузите с [cli.github.com](https://cli.github.com)

### Аутентификация не удалась
Выполните `gh auth login` снова и убедитесь, что завершили шаг браузера. Вам нужно ввести код и кликнуть "Authorize."

### "Repository already exists"
Repo с таким именем уже существует на вашем GitHub. Либо:
- Выбрать другое имя
- Удалить существующий repo на github.com сначала

### "Permission denied"
Ваш аутентификационный токен истёк. Перезапустите `gh auth login`.

### Не могу найти repo на github.com
Убедитесь, что вы вошли в правильный GitHub аккаунт в вашем браузере.
