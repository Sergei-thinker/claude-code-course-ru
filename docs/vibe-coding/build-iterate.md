# 4.3: Build и итерация

- **Время на выполнение:** 30-40 минут

> **Интерактивный урок:** Выполните `/start-4-3` в Claude Code

**В этом уроке:** Claude создаёт scaffold проекта, строит ваше приложение, и вы итерируете со скриншотами.

## Что такое сервер разработки?

Сервер разработки запускает ваш код локально, чтобы вы могли видеть его как веб-сайт.

Ключевые вещи, которые нужно знать:
- Он работает только на ВАШЕМ компьютере (не публичный)
- Позволяет тестировать и итерировать перед go live
- Должен продолжать работать, пока вы просматриваете сайт
- Остановить с `Ctrl+C` в терминале

## Что такое Localhost?

`localhost` означает ваш собственный компьютер, действующий как сервер.

`localhost:3000` = порт 3000 на вашей машине.

Это НЕ реальный URL. Никто другой не может получить к нему доступ. Вы получите реальный URL, когда деплоите в уроке 4.5.

## Что такое Plan Mode?

Plan mode - функция Claude Code, где Claude показывает вам план перед выполнением.

- Переключить с **Shift+Tab**
- Хорошо для больших задач, где вы хотите проверить сначала
- Вы одобряете план, затем Claude строит

Для сложных builds это позволяет задавать вопросы и делать корректировки перед тем, как написан код.

## Почему итерация важна

Первая версия никогда не идеальна. Это нормально и ожидаемо.

Профессиональная разработка всегда: построить → проверить → уточнить → повторить.

Разница с ИИ: итерации занимают секунды вместо дней. Так что не соглашайтесь - продолжайте уточнять, пока не полюбите.

## Trick со скриншотом

**Ctrl+V** (не Cmd+V на Mac!) вставляет изображения напрямую в Claude Code.

Это мощно для итерации:
- Сделать скриншот того, что неправильно
- Обвести проблему или указать на неё
- Вставить и описать, что вы хотите иначе

Визуальная обратная связь быстрее, чем описание проблем словами.

## Устранение неполадок

### "npm: command not found"
Node.js не установлен. Загрузите с [nodejs.org](https://nodejs.org), установите, затем перезапустите терминал.

### "Port 3000 already in use"
Другое приложение использует этот порт. Либо:
- Закрыть другие окна терминала, которые могут запускать серверы
- Использовать другой порт (Claude может помочь с этим)

### Браузер не открылся
Вручную перейдите на [http://localhost:3000](http://localhost:3000)

### Пустая страница
1. Проверьте ваш терминал на build ошибки (красный текст)
2. Проверьте консоль браузера: нажмите F12, кликните вкладку "Console", ищите красные ошибки

### Изменения не показываются
- Обновили браузер?
- Сервер разработки всё ещё работает в терминале?

### Стиль выглядит неправильно
Очистите кэш браузера с **Cmd+Shift+R** (Mac) или **Ctrl+Shift+R** (Windows), или попробуйте incognito окно.
