- **Время на выполнение:** 60-90 минут
- **Необходимые знания:** Модуль 1.4 (Оркестрация задач), Модуль 1.5 (Пользовательские субагенты)

> **Начните этот модуль в Claude Code:** Выполните `/start-2-3` для запуска интерактивного обучения.

## Обзор

Модуль 2.3 научит вас разрабатывать продуктовую стратегию, используя ИИ как партнёра для размышлений. Вы будете использовать Strategy Kernel фреймворк Румелта (Diagnosis → Guiding Policy → Coherent Actions) для структурирования стратегического мышления, проверять выборы с адвокатом дьявола и трансформировать стратегические документы в презентации для руководства.

**Ключевой вывод:** ИИ не может принимать стратегические решения за вас, но может помочь исследовать быстрее, применять фреймворки строго, оспаривать предположения и общаться чётко.

## Что такое продуктовая стратегия?

Продуктовая стратегия - это набор **сложных выборов** о том, где сфокусировать ресурсы и как создать конкурентное преимущество.

### Стратегия vs Не стратегия

| Не стратегия | Настоящая стратегия |
|--------------|---------------|
| "Увеличить доход на 50%" (цель) | "Фокус исключительно на voice-first ИИ для SMB, явно НЕ конкурируя по широте с Notion" (чёткие выборы + компромиссы) |
| "Построить ИИ чат, голос, автоматизацию" (функции) | "Углубиться в одну способность, где мы можем уникально победить с ограниченными ресурсами" (фокусированное направление) |
| "Быть лучшим инструментом продуктивности" (видение) | "Субсидировать стоимость ИИ изначально для построения защищённости через данные использования и retention" (позиционирование) |

### Роль ИИ в стратегии

| ИИ не может | ИИ может |
|-------------|-----------|
| Делать компромиссы (требует суждения) | Исследовать конкурентов параллельно |
| Предоставлять глубокий контекст о вашем рынке | Применять фреймворки строго |
| Брать на себя обязательства по сложным выборам | Оспаривать предположения (адвокат дьявола) |
| | Синтезировать документы и создавать презентации |

## Strategy Kernel фреймворк Румелта

### Трёхчастная структура

```
┌─────────────────────────────────────────┐
│         1. ДИАГНОЗ                      │
│    В чём вызов/возможность?             │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│       2. НАПРАВЛЯЮЩАЯ ПОЛИТИКА          │
│   Каков наш общий подход?               │
│   (ГДЕ конкурировать, КАК побеждать)    │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│      3. СОГЛАСОВАННЫЕ ДЕЙСТВИЯ          │
│   Какие конкретные шаги мы предпримем?  │
│   (Инициативы, которые усиливают друг   │
│    друга и соответствуют политике)      │
└─────────────────────────────────────────┘
```

### 1. Диагноз

Чёткое, действенное заявление о стратегическом вызове или возможности на основе реальных данных.

**Хороший диагноз:**
> ИИ ландшафт быстро накаляется. Notion, Linear и Asana запустили ИИ функции за последние 6 месяцев. Как меньший игрок с 2-человечной ИИ командой и ограниченным бюджетом (~$50k/квартал), мы не можем конкурировать по широте. Нам нужно найти защищённую позицию, где мы можем уникально победить с ограниченными ресурсами.

**Плохой диагноз:**
> Конкуренты делают ИИ (слишком расплывчато, нет инсайта)

### 2. Направляющая политика

Ваш общий подход, который делает **явные компромиссы** о том, где конкурировать и как побеждать.

**Хорошая направляющая политика:**
> Мы будем фокусироваться исключительно на voice-first ИИ для SMB (команды 5-20 человек), явно НЕ конкурируя по широте с более крупными игроками. Мы будем дифференцироваться через ИИ функции, которые нужны малым командам, но которые не будут строить конкуренты, ориентированные на enterprise. Мы будем субсидировать стоимость ИИ в базовом ценообразовании для стимулирования принятия, принимая меньшие маржи изначально для построения защищённости через данные использования и retention.

**Плохая направляющая политика:**
> Мы построим отличные ИИ функции и победим на рынке (нет выборов, нет компромиссов)

### 3. Согласованные действия

Конкретные, скоординированные инициативы, которые реализуют вашу направляющую политику и усиливают друг друга.

**Хорошие согласованные действия:**
```
Q1 2026:
- Расширить голос на заметки встреч + разбивку задач (углубление голосовой способности)
- Запустить SMB-специфичные шаблоны (workflow команд 5-20 человек)
- A/B тест ИИ в базовой цене vs premium tier

Q2 2026:
- Функции голосовой коллаборации (мультипользовательские голосовые сессии)
- SMB админ панель (отслеживание использования для лидов малых команд)
- Уточнить ценообразование на основе данных Q1
```

## Workflow разработки стратегии

```
1. КОНКУРЕНТНОЕ ИССЛЕДОВАНИЕ
   ↓ (Использовать параллельных агентов с WebSearch)

2. СТРАТЕГИЧЕСКИЕ ВЫБОРЫ
   ↓ (Сделать 5 ключевых решений о компромиссах)

3. АДВОКАТ ДЬЯВОЛА
   ↓ (Проверить давлением каждый выбор)

4. СИНТЕЗ
   ↓ (ИИ создаёт стратегический документ, используя Kernel Румелта)

5. ПРЕЗЕНТАЦИЯ
   ↓ (Использовать pptx skill для слайдов для руководства)
```

### 1. Конкурентное исследование с параллельными агентами

Запустите несколько агентов параллельно для одновременного исследования разных конкурентов.

```markdown
Запусти 3 агента для исследования конкурентного ИИ ландшафта:
- Агент 1: Исследуй ИИ функции Notion, ценообразование и позиционирование
- Агент 2: Исследуй ИИ функции Linear, ценообразование и позиционирование
- Агент 3: Исследуй ИИ функции Asana, ценообразование и позиционирование

Для каждого конкурента мне нужно:
- Какие ИИ функции они запустили
- Как они позиционируют и ценят ИИ
- Что кажется работает vs не работает
- Где может быть whitespace для нас
```

**Почему параллельные агенты работают:** В 3 раза быстрее, чем последовательное исследование, каждый агент использует WebSearch для последней информации.

### 2. Принятие стратегических выборов

Ответьте на 5 вопросов с множественным выбором, которые заставляют делать реальные компромиссы:

| Выбор | Опции |
|--------|---------|
| **#1: Фокус vs Широта** | A) Углубиться в одну способность<br/>B) Распределить ИИ по всем функциям<br/>C) Партнёрство для способностей |
| **#2: Конкурентный ответ** | A) Гонка по out-инновациям<br/>B) Дифференцироваться иначе<br/>C) Игнорировать и фокусироваться на roadmap |
| **#3: Бизнес-модель** | A) Premium ИИ tier (маржи важнее принятия)<br/>B) Субсидировать для стимулирования принятия<br/>C) Usage-based ценообразование |
| **#4: Продуктовый скоуп** | A) ИИ как продукт<br/>B) ИИ как улучшение<br/>C) ИИ для конкретных работ |
| **#5: Толерантность к риску** | A) Двигаться быстро, брать риски<br/>B) Обдуманно и защищённо<br/>C) Подождать и учиться |

Ваши 5 ответов становятся вашей Направляющей политикой.

### 3. Адвокат дьявола - Проверка давлением выборов

После каждого выбора используйте ИИ для оспаривания вашего мышления:

```markdown
Пользователь выбирает: A) Углубиться в голос

Ответ адвоката дьявола:
Вы ставите всё на голос. Но что если голос станет
коммодитизированным через 6 месяцев? OpenAI только что выпустил
лучшее распознавание голоса за 1/10 стоимости. Если у всех есть голос,
вы построили свою стратегию на чём-то, что больше не дифференцировано.
С 2-человечной командой у вас не будет ресурсов для pivot.
Разве это не рискованно?

Пользователь затем: Придерживаться выбора ИЛИ Пересмотреть
```

**Почему это работает:** Заставляет вас продумать минусы ПЕРЕД обязательством. Лучше услышать сложные вопросы от ИИ, чем от вашего CEO.

### 4. Синтез стратегических документов

Когда вы сделали ваши 5 стратегических выборов, ИИ синтезирует их в полный стратегический документ, используя Kernel Румелта:

**ИИ берёт:**
- Конкурентное исследование → Диагноз
- Ваши 5 выборов → Направляющая политика
- Создаёт 6-месячный roadmap, соответствующий выборам → Согласованные действия

**Вы предоставляете:**
- Стратегические выборы (ИИ не может их сделать)
- Контекст об ограничениях и ситуации
- Суждение о том, какие вызовы адвоката дьявола важны

**Структура вывода:**

```markdown
# H1 2026 ИИ продуктовая стратегия

## ДИАГНОЗ: Стратегический вызов
[Конкурентный ландшафт + ваши ограничения → основной вызов]

## НАПРАВЛЯЮЩАЯ ПОЛИТИКА: Наш стратегический подход
[Ваши 5 выборов, синтезированные в согласованное направление]
[Явные компромиссы - что мы НЕ делаем]

## СОГЛАСОВАННЫЕ ДЕЙСТВИЯ: H1 2026 Roadmap

### Q1 2026
- Инициатива 1 (соответствует выборам)
- Инициатива 2 (соответствует выборам)

### Q2 2026
- Инициатива 3 (основывается на Q1)

### Метрики успеха
- Метрика 1: [Цель]

## КРИТИЧЕСКИЕ ПРЕДПОЛОЖЕНИЯ
- Предположение 1: [Что должно быть правдой]

## КОНКУРЕНТНОЕ ПОЗИЦИОНИРОВАНИЕ
- Почему клиенты выбирают нас vs альтернативы
- Риски и смягчение
```

### 5. Создание презентаций для руководства со Skills

Трансформируйте ваш стратегический документ в слайды презентации, используя pptx skill:

```markdown
Используй pptx skill для создания презентации для руководства из
моего стратегического документа.

Требования:
- 12-15 слайдов, покрывающих полную стратегию
- Включить: Титульный, Executive Summary, Диагноз, Конкурентный
  ландшафт, Стратегическое направление, Компромиссы, Roadmap (Q1/Q2),
  Метрики успеха, Критические предположения, Почему мы победим, Риски
- Профессиональный дизайн для аудитории руководства
- Сохранить как strategy-review-slides.pptx
```

**Примечание:** pptx skill требует библиотеку `python-pptx`. Если установка не удаётся, вы можете создать markdown набросок слайдов вместо этого.

## Реальный пример

**Сценарий:** Вы Gen ИИ PM в TaskFlow. Вы запустили ИИ голосовой чат для задач (хорошо принят, умеренное использование). Руководство спрашивает: Как нам эволюционировать нашу ИИ стратегию для H1 2026?

**Подход:**

**1. Исследовать конкурентов параллельно:**
```markdown
Запусти 3 агента для исследования:
- Notion AI (позиционирование, функции, ценообразование)
- Linear AI (инженерно-ориентированные функции)
- Asana AI (enterprise фокус, стратегическое направление)
```

**2. Идентифицировать диагноз:**
На основе конкурентного исследования + ограничений (2-человечная ИИ команда, ~$50k/квартал бюджет):
> Мы не можем конкурировать по широте с ограниченными ресурсами. Нужно найти защищённую позицию, где мы уникально победим.

**3. Сделать стратегические выборы:**
- Фокус: Углубиться в голос (не широта)
- Конкуренция: Дифференцироваться для SMB
- Ценообразование: Субсидировать для стимулирования принятия
- Скоуп: ИИ как улучшение
- Риск: Обдуманно и защищённо

**4. Адвокат дьявола для каждого выбора:**
После каждого выбора услышать вызов и уточнить или придерживаться его.

**5. Синтезировать стратегию:**
ИИ создаёт полный стратегический документ с Диагнозом, Направляющей политикой, Согласованными действиями.

**6. Создать слайды:**
Использовать pptx skill для генерации 13-слайдовой презентации для руководства.

## Лучшие практики

**Делать:**
- Основывать диагноз на реальных данных (конкурентное исследование, обратная связь пользователей, рыночные тренды)
- Делать явные компромиссы - говорить, что вы НЕ делаете
- Использовать адвоката дьявола строго для проверки давлением каждого выбора
- Убедиться, что согласованные действия усиливают друг друга
- Документировать предположения и пересматривать ежеквартально

**Не делать:**
- Позволять ИИ принимать стратегические решения (это ваша работа)
- Путать цели со стратегией ("увеличить доход на 50%" не стратегия)
- Пропускать сложные компромиссы (настоящая стратегия требует говорить НЕТ)
- Игнорировать конкурентный ландшафт или set-and-forget стратегию

## Устранение неполадок

**Моя стратегия кажется общей - может применяться к любой компании**

**Решение:**
- Пересмотреть направляющую политику - говорит ли она, что вы НЕ делаете?
- Добавить конкретные ограничения: "С нашей 2-человечной командой и $50k бюджетом..."
- Тест: Мог бы конкурент скопировать это слово в слово? Если да, это слишком общее.

**Вопросы адвоката дьявола кажутся несправедливыми/чрезмерно негативными**

**Помните:** Лучше услышать сложные вопросы от ИИ, чем от вашего CEO. Если вы не можете защитить свой выбор, возможно, пересмотрите. Используйте вызовы для уточнения вашего мышления.

**pptx skill не установится / ошибки python-pptx**

**Решение:**
1. Попробуйте: `pip install python-pptx`
2. Если это не работает: `pip3 install python-pptx`
3. Если всё ещё не работает, создайте markdown набросок слайдов вместо этого

**Мои 5 стратегических выборов противоречат друг другу**

**Решение:**
- Пересмотрите все 5 выборов вместе - рассказывают ли они согласованную историю?
- Проверьте противоречия: Не может быть глубокий фокус И распределение везде
- Тест когерентности: Могли бы вы объяснить все 5 выборов в одном параграфе, который имеет смысл?
