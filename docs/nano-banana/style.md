# 3.1.3: Постоянство и стиль

- **Время на выполнение:** 20 минут
- **Необходимые знания:** Понимание основ (Модуль 3.1.2)

> **Начните этот модуль в Claude Code:** Выполните `/start-3-1-3` для запуска интерактивного обучения.

## Обзор

Модуль 3.1.3 научит вас писать промпты, которые получают потрясающие результаты и поддерживать постоянство через генерации. Вы узнаете Золотые правила промптинга от Google, как использовать референсные изображения и техники для исследования множественных направлений.

**Ключевой вывод:** Отличные результаты приходят от чёткой коммуникации, а не волшебных ключевых слов. Говорите с Gemini, как бы вы инструктировали человека-дизайнера.

## Золотые правила промптинга

Google выпустил официальное руководство для промптинга Gemini генерации изображений. Эти четыре правила трансформируют ваши результаты.

### Правило 1: Редактировать, не перебрасывать

**Если изображение на 80% правильно, запрашивайте конкретные изменения вместо начала заново.**

Это самое важное правило. Gemini - thinking model - она поддерживает контекст и понимает ваше творческое направление. Когда вы итерируете, она основывается на том, что работает.

**Вместо:**
```
[Сгенерировать изображение]
"Не совсем правильно"
[Сгенерировать снова с нуля]
[Сгенерировать снова с нуля]
[Сгенерировать снова с нуля]
```

**Делать так:**
```
[Сгенерировать изображение]
"Сделай освещение теплее"
"Переместить субъект влево"
"Добавь больше контраста"
[Готово]
```

**Почему это работает:** Каждое уточнение даёт Gemini больше информации о том, что вы хотите. Начало заново выбрасывает весь этот контекст.

### Правило 2: Использовать естественный язык и полные предложения

**Говорите с Gemini, как вы инструктируете человека-художника, а не пишете поисковые ключевые слова.**

Вы можете видеть "tag soup" промпты онлайн - ключевые слова, разделённые запятыми, как поисковый запрос. Это устаревший подход. Gemini - thinking model, которая понимает естественный язык.

**Tag soup (устаревший):**
```
профессиональный headshot, деловая одежда, уверенный, студийное освещение,
bokeh фон, острый фокус, 8k, trending на artstation
```

**Естественный язык (лучше):**
```
Профессиональный headshot уверенного бизнес руководителя в
современной студийной обстановке. Мягкое, ровное освещение создаёт тёплую атмосферу.
Фон имеет тонкое размытие, которое держит фокус на субъекте.
```

**Почему это работает:** Полные предложения предоставляют контекст и отношения между элементами. Gemini понимает "мягкое освещение, которое создаёт тепло" лучше, чем "мягкое освещение, тёплый."

### Правило 3: Быть конкретным и описательным

**Определите субъект, обстановку, освещение, настроение, текстуры и материалы. Углубляйтесь.**

Gemini может обрабатывать замечательные детали - гораздо больше, чем вы можете ожидать. Чем конкретнее вы, тем ближе вывод соответствует вашему видению.

**Расплывчато:**
```
Кофейня
```

**Конкретно:**
```
Уютный уголок кофейни дождливым днём. Тёплое освещение лампами Эдисона
отбрасывает золотые лужи на изношенные деревянные столы. Пар поднимается от
керамических кружек. Дождь течёт по окнам от пола до потолка. Несколько
посетителей читают книги в переполненных кожаных креслах. Стены - открытый кирпич
с винтажными концертными плакатами. Мягкий джаз играет из невидимого динамика.
```

**Элементы для рассмотрения:**
- **Субъект:** Кто/что в фокусе?
- **Обстановка:** Где это происходит?
- **Освещение:** Естественное, искусственное, время дня, направление, качество
- **Настроение:** Эмоциональный тон, атмосфера
- **Текстуры:** Качества поверхности, материалы
- **Цвета:** Палитра, контрасты, температура
- **Композиция:** Кадрирование, перспектива, фокальные точки

**Профессиональный совет:** Если не уверены, как добавить деталь, спросите Claude: "Помоги мне расширить этот промпт с большей конкретностью."

### Правило 4: Предоставить контекст

**Скажите Gemini "почему" или "для кого", чтобы она делала умнее творческие решения.**

Контекст формирует творческие выборы. Портрет "для детской книги" выглядит совершенно иначе, чем тот же субъект "для люксовой брендовой кампании."

**Без контекста:**
```
Лев в саванне
```

**С контекстом:**
```
Лев в саванне, для титульной карточки документального фильма о природе.
Изображение должно ощущаться величественным и кинематографичным, захватывая
силу и достоинство дикой природы.
```

**Примеры контекста:**
- "для pitch deck инвесторам"
- "для детского образовательного приложения"
- "для landing page премиум подписки"
- "в стиле Apple product маркетинга"
- "целевая аудитория - менеджеры производственных операций"

**Почему это работает:** Gemini использует контекст для подходящих выборов о стиле, тоне, композиции и уровне детализации.

## Референсные изображения

Референсные изображения - ваше секретное оружие для постоянства и контроля стиля. Вы можете предоставлять изображения для Gemini использовать как визуальный вход.

### Типы референсных изображений

| Тип | Цель | Пример |
|------|---------|---------|
| **Референс стиля** | Захватить визуальную эстетику | Дизайн landing page, художественный стиль |
| **Референс субъекта** | Поддерживать постоянство персонажа | Фото человека, продуктовая съёмка |
| **Референс композиции** | Направлять лэйаут и кадрирование | Wireframe, скриншот |

### Единый референс стиля

Предоставьте одно изображение для захвата его визуального стиля:

```
Сгенерируй [ваш субъект] в стиле этого референсного изображения
```

Gemini анализирует цвета референса, освещение, композицию и художественный подход - затем применяет этот стиль к вашему новому субъекту.

**Случаи использования:**
- Воссоздать визуальный язык бренда
- Соответствовать существующей кампании эстетике
- Достичь конкретного художественного стиля

### Множественные референсы субъекта

Для постоянных персонажей предоставьте несколько фото одного и того же субъекта:

```
Сгенерируй [промпт], используя эти референсные фото [субъекта]
```

**Профессиональный совет:** Больше референсных фото = лучшие результаты. 3-5 фото с разных углов и условий освещения помогают Gemini понять субъект полностью.

**Случаи использования:**
- Портреты персон, которые выглядят как конкретный человек
- Продуктовые съёмки с разных углов
- Постоянство персонажа через множественные изображения

### Микс и совмещение

Комбинируйте референсы стиля с референсами субъекта:

```
Создай [субъект из референса A] в стиле [референса B]
```

Это позволяет помещать реальные субъекты в художественные стили или применять брендовую эстетику к новому контенту.

**Пример из курса:**
- Референс стиля: Яркая баскетбольная landing page
- Референсы субъектов: Фото двух кошек (Winter и Piper)
- Результат: "APEX CAT" landing page в баскетбольном стиле, показывающая настоящих кошек

## Сетки: Множественные виды в одном изображении

Сетки генерируют несколько связанных изображений в одном выводе - идеально для листов персонажей, видов продукта или слайдов презентации.

### Когда использовать сетки

| Случай использования | Формат сетки | Пример |
|----------|-------------|---------|
| Дизайн персонажа | 3x3 позы | Лист спрайтов видеоигры |
| Виды продукта | 2x2 углы | E-commerce продуктовые съёмки |
| Презентация | 2x4 слайды | Учебные материалы |
| UI концепты | 2x3 экраны | Исследование дизайна приложения |

### Как запросить сетки

Будьте явны о структуре сетки:

```
Создай сетку 3x3, показывающую [субъект] в 9 разных позах:
- Ряд 1: Стоит, идёт, бежит
- Ряд 2: Сидит, прыгает, машет
- Ряд 3: Думает, указывает, празднует
```

**Ключевые советы:**
- Указать точные размеры сетки (3x3, 2x4, и т.д.)
- Описать, что каждая ячейка должна содержать
- Запросить постоянный стиль через все ячейки

### Случаи использования сеток для PM

- **Выражения персон:** Тот же персонаж, показывающий разные эмоции
- **Этапы user journey:** Визуальное представление каждого шага
- **Скриншоты функций:** Несколько экранов приложения в одном изображении
- **Слайды презентации:** Визуализация полного slide deck

## Варианты: Исследование направлений

Вместо обязательства по одному направлению, генерируйте несколько вариантов и выбирайте лучший.

### Workflow вариантов

1. **Сгенерировать 2-3 варианта** одной концепции
2. **Проверить все варианты** - они будут иметь значимо разные интерпретации
3. **Выбрать любимый**
4. **Итерировать на нём**, используя техники из Модуля 3.1.2

```
Сгенерируй 3 варианта [концепции] с разными творческими интерпретациями
```

### Когда использовать варианты

- **Неуверенны в стиле:** Позволить Gemini показать вам варианты
- **Презентации клиентам:** Предложить несколько направлений
- **Творческое исследование:** Посмотреть, что возможно
- **Преодоление блоков:** Когда не уверены, что хотите

### Варианты vs Итерация

| Варианты | Итерация |
|----------|-----------|
| Исследовать разные направления | Уточнять одно направление |
| Генерировать несколько сразу | Строить шаг за шагом |
| Выбрать, затем итерировать | Редактировать, не перебрасывать |
| Хорошо для: исследования | Хорошо для: уточнения |

**Лучшая практика:** Используйте варианты для нахождения вашего направления, затем переключитесь на итерацию для его совершенствования.

## Всё вместе

Вот полный workflow, комбинирующий все техники:


- Референс стиля желаемого вида

### Шаг 3: Сгенерировать варианты (если исследуете)

```
Сгенерируй 3 варианта с разными композициями
```

### Шаг 4: Выбрать и итерировать (Правило 1)

```
Мне нравится вариант 2. Сделай освещение теплее и пусть он смотрит
немного влево.
```

```
Идеально. Регенерируй в 2K разрешении для финальной версии.
```

## Лучшие практики

### Делать:
- **Писать полными предложениями** - коммуницировать естественно
- **Быть конкретным в деталях** - Gemini может обработать это
- **Предоставлять контекст** - объяснять цель
- **Использовать референсные изображения** - для постоянства стиля и субъекта
- **Итерировать терпеливо** - уточнять, а не перезапускать
- **Генерировать варианты** - при исследовании направлений

### Не делать:
- **Не использовать tag soup** - это устаревшее
- **Не быть расплывчатым** - конкретность получает результаты
- **Не пропускать контекст** - "почему" важно
- **Не перезапускать постоянно** - редактировать то, что работает
- **Не делать много изменений сразу** - по одному за раз

## Быстрая справка
