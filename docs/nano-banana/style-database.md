# 3.1.4: Создание вашей базы стилей

- **Время на выполнение:** 20 минут
- **Необходимые знания:** Постоянство и стиль (Модуль 3.1.3)

> **Начните этот модуль в Claude Code:** Выполните `/start-3-1-4` для запуска интерактивного обучения.

## Обзор

Модуль 3.1.4 обучает мета-навыку генерации изображений: построению личного творческого инструментария, который становится более ценным с каждым проектом. Вместо начала с нуля каждый раз, вы построите библиотеку проверенных стилей, которые можете применять мгновенно.

**Ключевой вывод:** Это не только о генерации изображений - это о построении компаундирующей системы. Каждый стиль, который вы сохраняете, делает будущую работу быстрее и лучше.

## Мета-навык

Большинство людей используют ИИ инструменты изображений как разовые новинки:
- Сгенерировать что-то
- Использовать раз
- Забыть об этом
- Начать заново в следующий раз

Опытные пользователи делают что-то другое:
- Сгенерировать что-то
- **Сохранить выигрышный промпт**
- Переиспользовать его между проектами
- **Строить библиотеку со временем**

Разница компаундируется драматически. После нескольких месяцев у вас библиотека проверенных стилей для каждой ситуации - персоны, диаграммы, презентации, маркетинг. Одна команда: "используй стиль #12" и готово.

## Ваша библиотека стилей

Модуль Nano Banana включает `style-library.html` - локальный HTML файл, который визуализирует вашу коллекцию стилей.

### Что в библиотеке

Каждая запись стиля содержит:

| Поле | Цель |
|-------|---------|
| **Номер индекса** | Быстрая ссылка ("используй стиль #02") |
| **Миниатюра** | Визуальный предпросмотр |
| **Название** | Описательное название |
| **Категория** | Фильтрация и организация |
| **Теги** | Поисковые ключевые слова |
| **Промпт** | Полный текст промпта |
| **Примеры использования** | Когда использовать этот стиль |

### Использование библиотеки

**Для применения стиля:**
```
Сгенерируй [ваш субъект], используя стиль #5
```

Claude ищет стиль, извлекает промпт и применяет его к вашему субъекту.

**Для просмотра стилей:**
Откройте `style-library.html` в вашем браузере. Используйте кнопки фильтра для исследования категорий.

### Категории

Библиотека организует стили в категории:

| Категория | Что содержит |
|----------|------------------|
| **Framework** | 2x2 матрицы, пирамиды, диаграммы Венна |
| **Flow** | Диаграммы процессов, journey maps, flowcharts |
| **Architecture** | Системные диаграммы, org charts, иерархии |
| **Mockup** | Wireframes, device frames, UI концепты |
| **Persona** | Портреты, lifestyle shots, headshots |
| **Marketing** | Реклама, анонсы, соцсети графика |
| **Artistic** | Иллюстрации, фотографические стили, ретро виды |

## Три способа роста вашей библиотеки

### Метод 1: Сохранять по ходу

Самый простой подход - когда создаёте что-то, что вам нравится, сохраняйте это.

**Workflow:**
1. Сгенерируйте изображение, которым вы довольны
2. Скажите Claude: "Добавь этот стиль в мою библиотеку"
3. Claude обновляет `style-library.html` с новой записью
4. Стиль теперь доступен для будущего использования

**Пример:**
```
Вы: Сгенерируй минималистичную journey map с мягкими цветами

[Claude генерирует изображение]

Вы: Мне нравится этот стиль. Добавь в мою библиотеку.

Claude: Готово! Добавлено как стиль #15 "Минималистичная Journey Map"
в категории Flow.
```

**Лучше всего для:** Захвата ваших собственных успешных экспериментов.

### Метод 2: Собирать из онлайна

Отличные промпты везде - соцсети, туториалы, базы промптов. Когда находите, тестируйте, затем сохраняйте.

**Workflow:**
1. Найдите многообещающий промпт онлайн
2. Тестируйте: "Сгенерируй [субъект], используя этот промпт: [вставить промпт]"
3. Если работает хорошо, скажите Claude сохранить
4. Теперь это часть вашей постоянной библиотеки

**Лучше всего для:** Курирования лучших промптов от сообщества.

### Метод 3: Извлечь из любого изображения

Это мощный ход. Видите изображение, которое любите? Извлеките его стиль как текст, затем воссоздавайте его когда угодно.

**Как это работает:**
1. Предоставьте изображение (путь к файлу или URL)
2. Claude запускает модуль извлечения стиля
3. Модуль анализирует: цвета, освещение, композицию, текстуры, настроение, эффекты
4. Возвращает детальное описание на естественном языке
5. Теперь вы можете воссоздавать этот стиль из чистого текста

**Workflow:**
```
Вы: Извлеки стиль из этого изображения: [путь к изображению]

Claude: [Запускает style_extract.py]

Вот извлечённый стиль:
"Тёплая аналоговая фотографическая эстетика с лёгким film grain.
Golden hour освещение создаёт мягкие тени. Приглушённые земляные тона
с desaturated зелёными и тёплыми highlights. Candid композиция
с лёгким motion blur, предполагающим authentic моменты..."

Вы: Сгенерируй портрет, используя тот извлечённый стиль

[Claude применяет стиль к новому субъекту]

Вы: Идеально. Сохрани это в мою библиотеку.
```

**Лучше всего для:**
- Захвата стилей из Pinterest, Instagram или дизайн вдохновения
- Воссоздания видов от конкурентов или брендов, которыми восхищаетесь
- Построения стиля из одного референсного изображения

### Модуль извлечения стиля

Модуль `style_extract.py` использует понимание изображений Gemini для анализа любого изображения и описания его визуального стиля.

**Что он захватывает:**
- Цветовую палитру и температуру
- Качество и направление освещения
- Композицию и кадрирование
- Текстуры и материалы
- Настроение и атмосферу
- Типографику (если присутствует)
- Специальные эффекты или обработку

**Почему это работает:** Gemini 3 Pro имеет исключительное понимание изображений. Она может артикулировать визуальные качества, которые было бы сложно описать вручную.

## Управление вашей библиотекой

### Добавление стилей

Когда вы говорите Claude сохранить стиль, она:
1. Назначает следующий доступный ID номер
2. Создаёт описательное название
3. Выбирает подходящую категорию
4. Добавляет релевантные теги
5. Сохраняет миниатюру изображения
6. Обновляет `style-library.html`

### Кастомизация библиотеки

Библиотека стилей - просто HTML файл - вы можете её кастомизировать:

- **Добавить колонки:** "Попроси Claude добавить колонку 'сложность'"
- **Изменить лэйаут:** "Реорганизуй библиотеку по категориям"
- **Добавить поиск:** "Добавь поисковую строку в библиотеку"
- **Модифицировать дизайн:** "Сделай библиотеку dark mode"

Это ВАШ инструментарий. Формируйте его под ваш workflow.

### Теги и фильтрация

Теги помогают быстро находить стили. Канонические теги по категориям:

| Категория | Доступные теги |
|----------|----------------|
| Framework | 2x2-matrix, pyramid, venn, canvas, concentric, triangle |
| Flow | process, journey-map, flowchart, steps, sequence |
| Architecture | hierarchy, hub-spoke, system-diagram, org-chart, tree |
| Mockup | wireframe, device-frame, ui-concept, landing-page, mobile, desktop |
| Persona | portrait, lifestyle, headshot, context, illustrated, scene |
| Marketing | ad, social, announcement, banner, hero |
| Artistic | flat-illustration, hand-drawn, watercolor, photography, retro, minimalist, bold-graphic, 3d-render |

Используйте кнопки фильтра в `style-library.html` для просмотра по категориям.

## Компаундирующий эффект

Вот почему это важно:

**Неделя 1:** У вас 5 стилей
- Базовое покрытие общих нужд

**Месяц 1:** У вас 15 стилей
- Паттерны появляются для вашей работы
- Начинаете иметь "любимые" стили

**Месяц 3:** У вас 30+ стилей
- Всеобъемлющее покрытие
- Редко нужно создавать с нуля
- "Использовать стиль #23" становится вашим workflow

**Год 1:** У вас 50+ проверенных в боях стилей
- Профессионального уровня библиотека
- Каждый стиль проверен через множественные проекты
- Минуты для производства того, что раньше занимало часы

Это компаундирующий эффект. Каждый проект делает вашу библиотеку более ценной.

## Примеры Workflow

### Пример 1: Постоянная визуализация бренда

**Сценарий:** Ваша компания имеет конкретный визуальный стиль, который вы должны поддерживать.

**Решение:**
1. Соберите 3-5 on-brand изображений
2. Извлеките стиль из вашего лучшего примера
3. Сохраните как "Company Brand Style"
4. Применяйте ко всей будущей графике

```
Сгенерируй анонс функции, используя стиль #8 (Company Brand)
```

### Пример 2: Быстрое исследование

**Сценарий:** Вам нужно 10 портретов user personas, все постоянные.

**Решение:**
1. Просмотрите вашу библиотеку для стилей персон
2. Выберите тот, который подходит тону проекта
3. Генерируйте все 10, используя тот же номер стиля

```
Сгенерируй портрет Ирины Соколовой, используя стиль #12
Сгенерируй портрет Артёма Николаева, используя стиль #12
Сгенерируй портрет Сергея Кузнецова, используя стиль #12
...
```

### Пример 3: Воссоздание вдохновения

**Сценарий:** Ваш CEO делится диаграммой конкурента, которая им нравится.

**Решение:**
1. Извлеките стиль из изображения конкурента
2. Сохраните в вашу библиотеку (с подходящим названием)
3. Примените к вашему собственному контенту

```
Извлеки стиль из этой диаграммы: [изображение конкурента]
Сгенерируй наш product roadmap, используя тот стиль
Сохрани это как "Executive Diagram Style"
```

## Лучшие практики

### Делать:
- **Сохранять стили часто** - если вам нравится, сохраняйте
- **Использовать описательные названия** - "Тёплый портрет" лучше, чем "Стиль 7"
- **Тегировать тщательно** - помогает будущему вам находить вещи
- **Извлекать агрессивно** - видите что-то хорошее? хватайте это
- **Периодически пересматривать** - удалять неиспользуемые стили

### Не делать:
- **Не копить посредственные стили** - качество важнее количества
- **Не забывать сохранять** - захватывать победы немедленно
- **Не перекатегоризировать** - держать просто
- **Не игнорировать извлечённые стили** - они часто лучшие
