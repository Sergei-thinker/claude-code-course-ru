# 3.2.1: Визуализация пользователей и продукта

- **Время на выполнение:** 25 минут
- **Необходимые знания:** Модуль 3.1 завершён (база стилей построена)

> **Начните этот модуль в Claude Code:** Выполните `/start-3-2-1` для запуска интерактивного обучения.

## Обзор

Модуль 3.2.1 применяет ваши навыки генерации изображений к реальным PM deliverables. Вы создадите визуальные ассеты, необходимые для продуктовых pitch: портреты персон, journey maps, wireframes, device mockups и героические изображения.

**Ключевой вывод:** Эти техники работают для любого pitch или презентации. Когда вы можете генерировать эти пять типов ассетов, вы можете визуально коммуницировать почти любую продуктовую историю.

## Сценарий

В Модуле 3.2 вы PM в TaskFlow (вымышленная B2B SaaS компания курса). Вы питчите **TaskFlow Mobile** - новое мобильное приложение для менеджеров операций в производстве.

Ваш pitch deck нуждается:
1. **Портрет персоны** - "Для кого мы строим?"
2. **Journey map** - "Какую проблему мы решаем?"
3. **Wireframe** - "Как выглядит решение?"
4. **Device mockup** - "Как это ощущается в руке?"
5. **Героическое изображение** - "Какое видение?"

Давайте построим каждое.

## Портреты персон

### Что они такое

Портреты персон придают лицо вашему целевому пользователю. Они делают абстрактные пользовательские сегменты реальными и помогают стейкхолдерам эмпатизировать с людьми, для которых вы строите.

### Когда использовать

- Слайды "Кто" в pitch deck
- Презентации исследований пользователей
- Секции персон в PRD
- Документы выравнивания команды

### Как генерировать

**Базовый промпт:**
```
Создай портрет персоны [имени], [роли] в [типе компании].
[Возрастной диапазон], [ключевые характеристики]. Профессионально, но приветливо.
Для pitch deck.
```

**Пример - Сергей Кузнецов:**
```
Создай портрет персоны Сергея Кузнецова, менеджера операций
на производственном заводе. Середина 40-х, практичный, уверенное, но
приветливое выражение. Носит рабочую одежду -
polo shirt, возможно, safety vest. Естественное освещение.
Для pitch deck руководству.
```

### Опции стилей

| Стиль | Лучше всего для | Пример добавления к промпту |
|-------|----------|------------------------|
| **Фотореалистичный** | Презентации руководству | "Фотореалистичный, профессиональный headshot качества" |
| **Иллюстрированный** | Дружественные/casual контексты | "Иллюстрированный в современном flat стиле" |
| **Стилизованный** | Специфичные для бренда нужды | Ссылайтесь на вашу библиотеку стилей |

### Профессиональные советы

- **Добавить контекстные подсказки:** Включить намёки среды (заводской цех, офис и т.д.)
- **Соответствовать тону:** Executive pitch = более формально; team wiki = более casual
- **Быть постоянным:** Если у вас несколько персон, использовать тот же стиль для всех

## Journey Maps

### Что они такое

Journey maps визуализируют опыт пользователя во времени - шаги, точки касания, болевые точки и эмоции через процесс.

### Когда использовать

- Слайды определения проблемы
- Находки исследований пользователей
- Фреймирование решения
- Выравнивание стейкхолдеров

### Как генерировать

**Базовый промпт:**
```
Создай journey map, показывающую [процесс/workflow пользователя].
Этапы: [список этапов]. Показать болевые точки на каждом этапе.
Чистый, готовый к презентации, 16:9 соотношение сторон.
```

**Пример - утро Сергея:**
```
Создай journey map, показывающую утренний workflow менеджера операций.
Этапы: Прибытие (6am) → Обзор бэклога → Утренний standup → Прогулка по цеху → Подготовка handoff.
Показать болевые точки: разрозненная информация, ручное отслеживание, задержанные обновления.
Чистый визуальный стиль, подходящий для pitch deck. 16:9 соотношение сторон.
```

### Опции стилей

| Стиль | Лучше всего для |
|-------|----------|
| **Минималистичный** | Презентации руководству |
| **Нарисованный от руки** | Workshop/collaborative ощущение |
| **Инфографический** | Маркетинг/внешнее использование |
| **Детальный** | Deep-dive презентации |

### Профессиональные советы

- **Включить эмоции:** Счастливые/фрустрированные лица на каждом этапе добавляют влияние
- **Выделять болевые точки:** Красные зоны или иконки привлекают внимание к проблемам
- **Держать сканируемым:** Макс 5-7 этапов; детали идут в поддерживающие документы

## Трансформация Wireframe

### Что это такое

Трансформируйте грубые наброски в полированные wireframes. Идеально для превращения набросков whiteboard с встреч в готовые к презентации.

### Когда использовать

- После brainstorming сессий
- Ранние ревью дизайна
- Быстрая валидация концептов
- Когда дизайнер недоступен

### Как генерировать

**Базовый промпт:**
```
Трансформируй этот нарисованный от руки набросок в чистый, полированный wireframe.
Сохрани тот же лэйаут и элементы. Профессиональное качество с
чистыми линиями и постоянным spacing.
```

Предоставьте ваш грубый набросок как референсное изображение.

**Пример:**
```
Трансформируй этот нарисованный от руки wireframe в полированный мобильный wireframe.
Сохрани лэйаут: header с приветствием, список задач, обзор смены,
нижняя навигация. Чистый, профессиональный, grayscale с хорошей типографикой.
Мобильное соотношение сторон (9:16).
```

### Профессиональные советы

- **Держать наброски чистыми:** Gemini нужно понять ваше намерение
- **Помечать элементы:** Писать текст в вашем наброске, чтобы Gemini знала, что куда идёт
- **Указывать fidelity:** "Low-fi wireframe" vs "high-fidelity mockup"

## Device Mockups

### Что они такое

Device mockups помещают ваши wireframes или скриншоты в фреймы телефонов/ноутбуков. Они делают цифровые продукты ощутимыми.

### Когда использовать

- Превью изображения app store
- Продуктовые слайды pitch deck
- Маркетинговые материалы
- Анонсы в соцсетях

### Как генерировать

**Базовый промпт:**
```
Помести этот wireframe в [device] mockup.
Чистый фон, лёгкая тень для глубины.
Профессионально, готов к презентации.
```

Предоставьте ваш wireframe как референсное изображение.

**Пример:**
```
Помести этот мобильный wireframe в iPhone 15 Pro mockup.
Чистый белый фон с тонкой тенью.
Лёгкий 3D угол для показа глубины. Профессиональное качество.
```

### Опции устройств

| Устройство | Лучше всего для |
|--------|----------|
| **iPhone** | Мобильные app pitches |
| **Android** | Кросс-платформенные или Android-first |
| **MacBook** | Веб приложения, панели управления |
| **iPad** | Планшетные опыты |
| **Множественные устройства** | Демонстрации responsive дизайна |

## Lifestyle Shots

### Что они такое

Lifestyle shots показывают ваш продукт в реальном использовании. Это "героические изображения", которые продают видение - человек, использующий ваш продукт в естественной среде.

### Когда использовать

- Обложки pitch deck
- Landing pages
- Маркетинговые героические изображения
- Анонсы в соцсетях

### Как генерировать

**Базовый промпт:**
```
[Персона], использующая [продукт] в [среде].
[Конкретные детали о контексте]. Естественное освещение.
Композиция подходит для [случая использования].
```

**Пример:**
```
Сергей Кузнецов, менеджер операций в середине 40-х, использует TaskFlow Mobile
на заводском цеху. Он уверенно держит телефон, проверяя
обновления задач. Производственная среда на фоне - оборудование,
работники, промышленная обстановка. Естественное освещение из складских окон.
Композиция работает как героическое изображение pitch deck.
```

### Советы по композиции

| Элемент | Руководство |
|---------|----------|
| **Размещение субъекта** | Правило третей; оставить пространство для текстового оверлея, если нужно |
| **Фон** | Релевантный, но не отвлекающий; добавляет контекст |
| **Освещение** | Естественное обычно ощущается более authentic |
| **Выражение** | Должно соответствовать истории (уверенный, сфокусированный, удовлетворённый) |

## Полный пример Pitch Deck

Вот как все пять ассетов работают вместе для pitch TaskFlow Mobile:

### Слайд 1: Герой
**Ассет:** Lifestyle shot Сергея, использующего TaskFlow Mobile на заводском цеху
**История:** "Это будущее, которое мы строим"

### Слайд 2: Пользователь
**Ассет:** Портрет персоны Сергея Кузнецова
**История:** "Встречайте Сергея - наш целевой пользователь"

### Слайд 3: Проблема
**Ассет:** Journey map хаотичного утра Сергея
**История:** "Его текущий workflow сломан"

### Слайд 4: Решение
**Ассет:** Полированный wireframe TaskFlow Mobile
**История:** "Вот что мы строим"

### Слайд 5: Видение
**Ассет:** Device mockup (iPhone с wireframe)
**История:** "Управление задачами в вашем кармане"

**Общее время:** 15-20 минут для всех пяти ассетов.

## Лучшие практики

### Делать:
- **Соответствовать стили через deck** - постоянство выглядит профессионально
- **Использовать контекст TaskFlow** - или ваш реальный контекст компании
- **Генерировать на 1K сначала** - итерировать, затем идти на 2K для финалов
- **Сохранять хорошие стили** - добавлять в вашу библиотеку
- **Генерировать варианты** - особенно для героических изображений

### Не делать:
- **Не смешивать дико разные стили** - выглядит непрофессионально
- **Не пропускать итерацию** - первые черновики редко финальные
- **Не переделывать wireframes** - они должны ощущаться как wireframes
- **Не забывать контекст** - говорить Claude, для чего это
